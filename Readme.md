# KWIC 单词移位排序系统（命令行 & GUI）

本项目实现了一个带 **命令行界面** 和 **图形界面** 的**KWIC（Key Word In Context，关键字在上下文中）** 单词移位排序与检索工具。

它可以对多行英文文本进行：

- 单词提取与预处理（带 URL 识别）
- 单词循环移位（KWIC 核心）
- 按自定义字母优先级排序（忽略 URL）
- 多关键词检索（支持短语）
- GUI 中支持文本输入、文件导入、搜索与结果导出

------

## 功能

1. **文本行输入与预处理**
   - 支持命令行逐行输入，或 GUI 中多行输入/单行输入/从文件导入。
   - 自动过滤非英文字母字符，仅保留 `A–Z / a–z` 与空格。
   - 自动识别每行中的 **最后一个 URL**（以 `http://` 或 `https://` 开头），并将其与文本部分分离保存。
   - 行内若既没有有效单词、也没有 URL，则视为无效行并被忽略。
2. **KWIC 单词循环移位**
   - 对每一行（仅文本部分）进行循环移位，生成多个“视图”：
     - 例如：`hello world example`
       - `hello world example`
       - `world example hello`
       - `example hello world`
   - 若原行含 URL，则每个移位结果末尾都会附带该 URL。
3. **排序（忽略 URL）**
   - 对所有移位结果进行统一排序：
     - 排序时会 **忽略 URL 字段**，只按单词本身排序。
     - 采用自定义大小写敏感优先级：`a < A < b < B < ...`。
4. **多关键词检索**
   - 命令行模式：在 `/sort` 输出后进入交互式检索模式，支持“每行一个关键词”输入。
   - GUI 模式：在“关键词搜索”标签页中输入多行关键词，一次性查询。
   - 支持带引号的短语搜索，如 `"machine learning"`。
   - 搜索时通过在文本前后补空格，以尽量实现 **整词匹配**。
5. **图形界面 (Tkinter GUI)**
   - 多标签页布局：
     - “输入区”：输入文本、导入文件、查看移位结果/排序结果、导出结果。
     - “结果区”：下含“移位结果 / 排序结果 / 关键词搜索”三个子标签页。
   - 文件导入：支持从 `.txt` 文件读取多行文本。
   - 结果导出：可将排序结果保存为文本文件（每行“序号 + 文本”）。
6. **命令行交互**
   - 输入普通文本行进行处理。
   - 输入 `/sort` 输出排序结果，然后进入多关键词检索模式。
   - 支持 `Ctrl + C` 中断程序。

## 项目结构 

```text
.
├─ main.py        # 命令行入口 & MainController
├─ gui.py         # Tkinter 图形界面入口
├─ input.py       # InputModule：解析行、提取单词和 URL
├─ line.py        # LineModule：存储行数据（单词列表 + URL）
├─ shift.py       # ShiftModule：生成每行的循环移位结果
├─ sort.py        # SortModule：根据自定义规则排序移位结果
├─ output.py      # OutputModule：打印/提供排序结果
└─ README.md      # 项目说明（即本文档）
```

### 各模块简介

- **`InputModule` (`input.py`)**
  - `process_input(line_str)`：拆分文本行、识别最后一个 URL、清洗非 URL 文本，只保留英文字母和空格；构造 `(words, url)` 并交给 `LineModule` 存储。
  - 记录哪些单词在原始字符串中带有“前导空格”（用于排序中区分）。
- **`LineModule` (`line.py`)**
  - 内部储存 `[(words, url), ...]`。
  - 支持通过行号获取：仅单词 / URL / 全部元组等。
- **`ShiftModule` (`shift.py`)**
  - 对每一行的 `words` 做循环移位，每种移位都附上同一个 URL（若存在）。
  - 提供：
    - `get_shift_results(line_number)`
    - `get_all_shifted_entries()` 返回 `(line_num, shift_index, shifted_words)` 列表。
- **`SortModule` (`sort.py`)**
  - 通过 `get_sorted_results()` 对所有移位结果进行排序，排序键 `_get_key(words)` 会先剔除 URL 再按自定义优先级比较各单词字符。
  - `get_alphabetical_sorted_words()` 可以得到“带字母优先级的所有唯一单词列表”，并考虑单词在原始文本中是否有前导空格。
- **`OutputModule` (`output.py`)**
  - 提供命令行打印移位结果和排序结果的函数。
  - `get_sorted_results()` 返回 `(连续编号, 文本字符串, 原始行号)` 的列表，供 GUI 使用。
- **`MainController` (`main.py`)**
  - 负责命令行交互逻辑，协调 `InputModule / ShiftModule / SortModule / OutputModule`。
  - 支持输入文本、执行 `/sort`，以及多关键词检索模式。
- **`KWICGUI` (`gui.py`)**
  - 通过 Tkinter 实现完整 GUI：多分页输入/输出、文件导入、结果导出、多关键词搜索。
  - 复用与命令行同样的底层模块（Line/Input/Shift/Sort/Output），保持逻辑一致。

## 使用方法 🧪

### 1. 克隆或下载项目

```bash
git clone <your-repo-url>
cd <your-project-folder>
```

（如果你是直接拷贝文件，只要把这些 `.py` 文件放在同一目录下即可。）

------

### 2. 启动命令行版本

```bash
python main.py
```

启动后，你会看到类似提示：

```text
欢迎使用单词移位排序系统!
输入文本行或命令。可用命令:
  /sort - 显示所有移位结果和排序结果
输入文本行进行排序处理。
```

#### 基本流程（CLI）

1. **输入若干行文本**（英文、可带 URL）：

   ```text
   > The quick brown fox jumps over the lazy dog
   > KWIC is a keyword in context index system https://example.com
   ```

2. **输入命令 `/sort` 生成排序结果**：

   ```text
   > /sort
   
   排序结果 (包含URL):
   1 ...
   2 ...
   ...
   -------------------结束-------------------
   ```

   排序结果中每一行是一个单词移位后的视图（若有 URL 则在最后）。

3. **进行多关键词检索**：

   - 在 `/sort` 之后自动进入：

     ```text
     ---------多关键词检索模式-------------
     提示：每行输入一个搜索关键词（支持带引号的短语）
     输入完成后输入 /search 开始检索
     输入 /exit 退出此模式
     ```

   - 示例：

     ```text
     (搜索词)>> quick
     (搜索词)>> "keyword in context"
     (搜索词)>> /search
     "quick": 1 3
     "keyword in context": 5
     再次输入关键词或命令...
     ```

   - 输入 `/exit` 退出检索模式并结束程序。

------

### 3. 启动图形界面版本（推荐体验 GUI 😎）

```bash
python gui.py
```

启动后，会打开一个名为 **“KWIC - 单词移位排序工具”** 的窗口。

#### GUI 主要区域说明

1. **主界面结构**

   - 顶部是一个 **Notebook 分页**：
     - “输入区”
     - “结果区”

2. **输入区**

   - **多行输入文本框**
     - 在“输入区域”中输入多行文本（每行单独处理），点击“添加文本”批量加入。
   - **文件导入**
     - “导入文件”区域中：
       - 点击“选择文件”选择 `.txt` 文本文件；
       - 点击“导入”将文件中的每一非空行加入处理。
   - **单行输入**
     - 在“输入单行文本”框中输入一行，点击“添加行”加入。
   - **处理按钮**
     - `显示移位结果`：在“结果区”的“移位结果”标签下显示所有按行的循环移位。
     - `显示排序结果`：在“排序结果”标签页中显示全局排序后的视图。
     - `导出结果`：将当前排序结果保存为 `.txt` 文件。

3. **结果区**

   “结果区”内部又是一个 Notebook，包含三个子标签页：

   - **“移位结果”**

     - 逐行显示每一原始行的所有移位结果，形式大致为：

       ```text
       移位结果 #1:
       1: ...
       2: ...
       ```

   - **“排序结果”**

     - 显示全局排序后的结果，每行带一个序号：

       ```text
       1 KWIC is a keyword in context ...
       2 The quick brown fox ...
       ```

   - **“关键词搜索”**

     - 上部是多行搜索输入框（每行一个关键词）。

     - 点击“搜索”后，会在下方文本框中显示类似：

       ```text
       "quick": 1 3
       "keyword in context": 未找到匹配
       ```

     - 底部的标签会同步显示搜索“汇总”。

4. **导出结果**

   - 点击“导出结果”按钮时，会弹出保存对话框，选择路径和文件名后，将排序结果以如下格式保存：

     ```text
     1\t<文本内容>
     2\t<文本内容>
     ...
     ```

## 输入与排序规则说明 📚

1. **输入内容规则**
   - 每一行中：
     - 最右侧的 `http://...` 或 `https://...` 被当作 URL，并单独保存；其左侧才是参与移位与排序的文本。
     - 文本部分会清洗成 **仅包含英文字母与空格** 的字符串（其他符号全部丢弃）。
   - 若一行中既没有 URL 又没有有效单词，则该行会被忽略。
2. **循环移位规则**
   - 仅对 `words` 列表进行循环移位；
   - 若存在 URL，则每个移位结果末尾都添加该 URL。
3. **排序规则（SortModule）**
   - 排序时先从移位结果中剔除 URL，然后按单词逐个比较；
   - 字符级优先级顺序为：
     - `a < A < b < B < ... < z < Z`；其它字符视为优先级无穷大。
   - 最终结果是一个按“忽略 URL 的字母序”排列的移位列表。
4. **关键词检索规则**
   - 所有检索都会将文本和关键词统一为小写；
   - 搜索时会在文本字符串前后补空格，匹配 `" keyword "` 形式，从而尽量贴近整词匹配： 
   - 支持短语：
     - 若关键词被双引号或单引号包裹，会先去掉引号再搜索，如：`"machine learning"`。
