# KWIC 单词循环移位排序系统

本仓库是“计算机软件基础 I”软件工程大作业项目，实现了一个基于 Python 的 KWIC（Key Word In Context，关键字上下文索引）单词循环移位排序系统。

项目支持命令行版本和图形界面版本，可以对英文文本进行输入、预处理、循环移位、排序、关键词检索以及结果导出。

## 项目简介

KWIC 是一种常见的文本索引方法。它会将每一行文本中的单词进行循环移位，并将所有移位结果按字典序排序，从而方便用户根据关键词快速定位相关文本。

例如输入：

```text
software engineering project
```

系统会生成如下循环移位结果：

```text
software engineering project
engineering project software
project software engineering
```

随后系统会对所有移位结果进行统一排序，并支持关键词检索。

## 主要功能

* 支持多行英文文本输入
* 支持从 `.txt` 文件导入文本
* 支持 KWIC 单词循环移位
* 支持对所有移位结果进行排序
* 支持关键词检索
* 支持短语检索，例如 `"software engineering"`
* 支持识别并保留文本末尾的 URL
* 支持命令行交互
* 支持 Tkinter 图形界面
* 支持将排序结果导出为 `.txt` 文件

## 模块说明

### `main.py`

命令行版本的程序入口，负责创建并协调各个模块。

主要功能包括：

* 接收用户输入的文本行
* 识别 `/sort` 命令
* 输出排序结果
* 进入关键词检索模式

### `gui.py`

图形界面版本的程序入口，使用 Python 标准库中的 `tkinter` 实现。

GUI 支持：

* 多行文本输入
* 单行文本输入
* 文本文件导入
* 显示移位结果
* 显示排序结果
* 关键词搜索
* 导出排序结果

### `input.py`

输入处理模块，负责对原始文本进行预处理。

主要处理逻辑：

* 分离文本中的 URL
* 保留每行中最后一个以 `http://` 或 `https://` 开头的 URL
* 对普通文本部分进行清洗
* 仅保留英文字母和空格
* 将处理后的单词列表传递给行管理模块

### `line.py`

行数据管理模块，用于保存输入后的文本数据。

每一行数据会被保存为：

```text
(words, url)
```

其中：

* `words` 表示单词列表
* `url` 表示该行对应的 URL，如果没有 URL，则为空

### `shift.py`

循环移位模块，负责对每一行文本生成 KWIC 移位结果。

例如：

```text
hello world example
```

会生成：

```text
hello world example
world example hello
example hello world
```

如果原文本末尾带有 URL，则 URL 会保留在每一条移位结果的末尾。

### `sort.py`

排序模块，负责对所有移位结果进行统一排序。

排序时会忽略 URL，只根据文本单词本身进行排序。

当前排序规则为大小写敏感排序，字符优先级为：

```text
a < A < b < B < c < C < ... < z < Z
```

### `output.py`

输出模块，负责整理并输出结果。

主要用于：

* 打印循环移位结果
* 打印排序结果
* 为 GUI 提供可显示、可导出的排序结果

## 运行环境

本项目使用 Python 编写，运行环境如下：

* Python 3.x
* Tkinter

本项目未使用第三方 Python 库，通常安装 Python 后即可直接运行。

如果运行 GUI 时提示缺少 Tkinter，需要根据系统环境安装对应组件。

## 使用方法

### 1. 克隆仓库

```bash
git clone https://github.com/Cassenlong/KWIC.git
cd KWIC/phase
```

注意：程序源码位于 `phase/` 文件夹中，因此运行程序前需要先进入该目录。

## 命令行版本

在 `phase/` 目录下运行：

```bash
python main.py
```

程序启动后会显示提示信息：

```text
欢迎使用单词移位排序系统!
输入文本行或命令。可用命令:
  /sort - 显示所有移位结果和排序结果
输入文本行进行排序处理。
```

### 示例输入

```text
> The quick brown fox
> KWIC is a keyword in context system https://example.com
> /sort
```

输入 `/sort` 后，系统会输出排序结果，并进入关键词检索模式。

### 关键词检索

进入检索模式后，可以每行输入一个关键词：

```text
(搜索词)>> quick
(搜索词)>> "keyword in context"
(搜索词)>> /search
```

系统会输出每个关键词匹配到的排序结果编号。

如果想退出检索模式，可以输入：

```text
/exit
```

## 图形界面版本

在 `phase/` 目录下运行：

```bash
python gui.py
```

运行后会打开名为：

```text
KWIC - 单词移位排序工具
```

的图形界面窗口。

图形界面主要包含两个区域：

* 输入区
* 结果区

### 输入区

输入区支持三种输入方式：

1. 在多行文本框中输入多行文本
2. 在单行输入框中添加一行文本
3. 从本地 `.txt` 文件导入文本

### 结果区

结果区包含三个子页面：

1. 移位结果
2. 排序结果
3. 关键词搜索

用户可以先添加文本，然后点击：

* `显示移位结果`
* `显示排序结果`
* `导出结果`

在关键词搜索页面中，可以输入一个或多个关键词，每行一个关键词，然后点击搜索。

## 输入规则说明

### 普通文本处理

系统主要面向英文文本处理。对于普通文本部分，程序会保留：

* 英文字母 `A-Z`
* 英文字母 `a-z`
* 空格

其他符号会在预处理阶段被过滤。

### URL 处理

如果一行文本中包含 URL，系统会识别该行中最后一个以如下内容开头的字符串：

```text
http://
https://
```

该 URL 不参与循环移位和排序，但会被保留在对应移位结果的末尾。

示例：

```text
hello world https://example.com
```

移位结果为：

```text
hello world https://example.com
world hello https://example.com
```

### 空行处理

如果某一行既没有有效单词，也没有 URL，该行会被忽略。

## 排序规则说明

系统会先生成所有行的循环移位结果，然后统一排序。

排序时：

* URL 不参与排序
* 单词按照字符优先级逐个比较
* 当前字符优先级为 `a < A < b < B < ... < z < Z`

因此，该系统不是简单调用 Python 默认字符串排序，而是使用了自定义排序规则。

## 搜索规则说明

关键词搜索支持：

* 单个单词搜索
* 多关键词搜索
* 带引号的短语搜索

示例：

```text
quick
"keyword in context"
```

搜索时会忽略大小写。

系统会在排序结果中查找匹配内容，并输出匹配到的排序结果编号。

## 结果导出

在 GUI 版本中，点击 `导出结果` 后，可以将当前排序结果保存为 `.txt` 文件。

导出格式大致如下：

```text
1    sorted result text
2    another sorted result text
3    ...
```
